# 관리자 계정 설정 가이드

## 첫 관리자 계정 생성

인증 시스템이 설정되었습니다. 첫 관리자 계정을 생성하려면 다음 단계를 따르세요:

### 방법 1: 웹 애플리케이션을 통한 회원가입 (권장)

1. 개발 서버 실행:
   ```bash
   npm run dev
   ```

2. 브라우저에서 회원가입 페이지로 이동:
   ```
   http://localhost:3000/signup
   ```

3. 다음 정보로 회원가입:
   - 이메일: `sungjunpark7392@gmail.com`
   - 비밀번호: `123456`
   - 이름: (선택사항)

4. **첫 사용자는 자동으로 관리자 역할이 부여됩니다.**

5. 회원가입 완료 후 로그인 페이지로 이동하여 로그인하세요.

### 방법 2: SQL을 통한 직접 생성 (고급)

만약 이미 사용자가 생성되어 있다면, SQL을 통해 역할을 업데이트할 수 있습니다:

```sql
-- 사용자 ID 확인
SELECT id, email FROM auth.users WHERE email = 'sungjunpark7392@gmail.com';

-- 프로필 역할 업데이트 (위에서 확인한 ID 사용)
UPDATE user_profiles 
SET role = 'admin' 
WHERE id = '사용자_ID';
```

## 확인 사항

회원가입 후 다음을 확인하세요:

1. **로그인 테스트**: `/login` 페이지에서 로그인 가능한지 확인
2. **프로필 확인**: Navigation 바에서 사용자 이름과 역할이 표시되는지 확인
3. **권한 확인**: 관리자 역할이 제대로 부여되었는지 확인

## 보안 주의사항

- 첫 관리자 계정 생성 후, 프로덕션 환경에서는 반드시 초대 시스템을 사용하세요.
- 현재 설정은 첫 사용자만 초대 없이 회원가입할 수 있습니다.
- 두 번째 사용자부터는 초대 토큰이 필요합니다.

## 초대 시스템 사용

관리자가 다른 사용자를 초대하려면:

1. 관리자로 로그인
2. 초대 생성 API 사용 (향후 관리자 페이지에서 구현 예정)
3. 초대 링크를 사용자에게 전달: `/signup?token={토큰}`

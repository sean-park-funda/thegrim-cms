# 스토리텔러 (Storyteller) 기능 설계

> 작가의 작은 개입으로 완성도 높은 스토리를 만드는 AI 창작 보조 시스템

---

## 1. 핵심 개념

작가가 짧은 문장을 던지면, AI가 작가의 문체로 확장해준다.

```
입력: "박형석이 발차기를 했다"
출력: "박형석의 오른발이 공기를 가르며 상대의 턱을 향해 뻗었다. 
      근육에 실린 수년간의 분노가 그 한 동작에 응축되어 있었다..."
```

작가는 스토리를 **직접 쓰는 것이 아니라 지휘**한다. 어떤 장면이 필요한지만 말하면 AI가 살을 붙인다. 작가는 결과물을 승인하거나, 방향을 수정하거나, 다시 생성한다.

---

## 2. 주요 기능 모듈

### 2.1 문체 학습 (Style Engine)

**설정 방법**
- 작가가 직접 쓴 글 샘플 업로드 (최소 2,000자 이상)
- 기존 원고 파일 연동 (프로젝트 내 스크립트)
- AI가 문체 특성 분석:
  - 문장 길이 패턴
  - 자주 쓰는 어휘/표현
  - 감정 묘사 방식
  - 액션 묘사 리듬감

**문체 카드 UI**
```
[내 문체 분석]
• 평균 문장 길이: 짧음 (15~20자)
• 특징: 끊어치는 리듬, 내면 독백 多, 직유보다 은유 선호
• 자주 쓰는 표현: "~하듯이", "그 순간", "아무도 몰랐지만"
[샘플 다시 보기] [추가 학습]
```

---

### 2.2 등장인물 관리 (Character Board)

#### 등장인물 생성 플로우

```
[+ 등장인물 추가] 클릭
  ↓
간단한 설명 입력:
"주인공. 30대 남자. 전직 형사. 냉정하지만 의리 있음"
  ↓
AI가 상세 프로필 자동 생성
```

**생성되는 프로필 항목**
| 항목 | 예시 |
|------|------|
| 이름 | 박형석 |
| 나이/성별 | 34세 남성 |
| 외모 | 180cm, 단단한 체형, 옅은 흉터가 왼쪽 눈썹 위에 있음... |
| 직업 | 전직 강력계 형사, 현재 무직 |
| 성격 | ISTJ, 말수 적음, 한 번 맡은 일은 끝까지 |
| 내면 심리 | 5년 전 놓친 사건에 대한 죄책감, 술로 달래는 중 |
| 말투 | 짧고 단호함. 감탄사 거의 없음 |
| 관계 | - |
| 핵심 동기 | 억울하게 죽은 후배의 원수를 갚는 것 |
| 비밀 | 그 사건 당일 자신도 현장에 있었다 |

**인물 카드 UI**
- 상단: 캐릭터 이미지 (CMS의 캐릭터 이미지 연동 또는 AI 생성)
- 핵심 정보 요약 배지 (성격 태그, 역할 등)
- [새로 생성] [세부 수정] [삭제] 버튼
- 각 항목 개별 수정 가능 (클릭 시 인라인 편집)

---

### 2.3 관계도 (Relationship Graph)

**인터랙티브 캔버스**
- 등장인물 카드를 캔버스에 배치
- 캐릭터 간 선을 드래그하여 연결
- 연결선 클릭 시 관계 유형 설정:

```
관계 유형:
● 가족 (부모/자녀/형제)
● 적대
● 연인/과거 연인
● 동료/파트너
● 일방적 감정 (짝사랑, 집착)
● 사제 관계
● 비밀 관계
● 직접 입력
```

- 관계에 메모 추가 가능: "3년 전 함께 사건 해결, 이후 연락 끊김"
- AI가 관계도를 보고 잠재적 갈등/플롯 포인트 제안

**관계도 자동 분석 예시**
```
[AI 분석] 
박형석 ↔ 이수진의 관계가 '과거 연인'이면서
이수진이 현재 '악당 오민준'과 연결되어 있습니다.
→ 삼각 갈등 플롯이 가능합니다. 플롯에 추가할까요?
```

### 2.4 스토리 채팅 (Story Chat)

**핵심 인터페이스**: 채팅 창처럼 생겼지만, 실제로는 스토리를 짜는 협업 공간

```
작가: 박형석이 발차기를 했다

AI: [생성됨]
박형석의 오른발이 허공을 가르며 상대의 턱을 향해 솟구쳤다.
근육 하나하나에 힘이 응집되는 순간이었다. 쿠웅— 둔탁한 충돌음.
상대는 말 한마디 못 하고 뒤로 꺾였다.
그는 숨을 고르지도 않았다. 이미 다음 목표를 보고 있었다.

[승인] [재생성] [수정 요청] [단어 선택 편집]
```

**수정 요청 예시**
```
작가: 좀 더 빠른 호흡으로. 문장 더 짧게.

AI: [재생성됨]
발이 솟구쳤다.
쿠웅—
상대가 꺾였다.
형석은 이미 다음을 보고 있었다.

[승인] [재생성] [수정 요청]
```

**장면 지시 방식**
- 한 줄 지시: "형석이 적을 제압했다"
- 감정 태그 첨부: "형석이 적을 제압했다 [분노:90%] [냉정함:60%]"
- 컷 번호 지정: "#14컷 — 형석이 멈춘다. 뭔가를 깨달은 표정"
- 대사 지정: "형석이 '거기서 기다려'라고 말하며 달려나갔다"

### 2.5 플롯 관리 (Plot Board)

**플롯 구조 시각화**

3단 구조 (또는 커스텀 구조) 선택:
```
[발단] → [전개] → [위기] → [절정] → [결말]
```

각 단계에 **플롯 카드** 배치:
- 카드 제목 (핵심 사건)
- 등장인물 태그
- 예상 분량 (화 수)
- 상태: 아이디어 / 작성중 / 완료

**AI 플롯 제안**
- 관계도 + 기존 플롯 카드 기반으로 AI가 다음 전개 3가지 제안
- 작가가 선택하거나, 아이디어만 차용

```
[AI 플롯 제안]
1. 이수진이 오민준의 스파이임이 밝혀지는 장면 삽입
   → 형석의 신뢰 붕괴 + 갈등 극대화 (3~5화 분량)

2. 형석이 과거 사건 현장으로 돌아가 새로운 단서 발견
   → 사건 미스터리 강화 (1~2화)

3. 새 조력자 등장 — 형석과 적대적으로 시작
   → 버디물 전개 가능 (장기 서브플롯)

[플롯에 추가] [아이디어 저장] [무시]
```

---

### 2.6 애셋 연동 (Asset Library)

스토리텔러 안에서 CMS의 기존 애셋과 연동:

**캐릭터 ↔ 이미지 연결**
- 인물 카드에 기존 캐릭터 디자인 시트 연결
- 관계도에서 실제 얼굴로 표시

**장면 ↔ 컷 연결**
- 스토리 채팅의 각 장면에 기존 컷 이미지 태그
- 해당 컷에서 어떤 대사/묘사가 쓰였는지 추적

**배경 아카이브**
- 자주 등장하는 배경 (학교, 골목, 옥탑방 등) 저장
- 묘사 시 AI가 해당 배경 설정을 자동 참조

```

├──────────┬──────────────────────────────────┬───────────────┤
│          │                                  │               │
│ 등장인물 │         스토리 채팅              │  플롯 보드    │
│  보드    │                                  │               │
│          │  작가: 형석이 문을 박찼다        │  [발단]       │

│ [수진]   │  AI: 쾅— 낡은 경첩이 비명을... │  [전개]       │
```

- 관계도 모드: 인물 보드 + 관계 캔버스 전체화면
- 플롯 모드: 플롯 보드 전체화면

## 4. 작가 개입 레벨 설정

```
[자동화 레벨]
1단계: 힌트만 (핵심 문장만 제안, 작가가 직접 씀)
2단계: 초안 제공 (완전한 문단 제공, 작가가 편집)  ← 기본
3단계: 연속 생성 (작가가 승인만 하면 계속 전개)
```

레벨에 따라 AI가 얼마나 주도적으로 이야기를 끌어가는지 달라짐.

## 5. 데이터 구조 (DB 설계 방향)

```
storyteller_projects
├── id, webtoon_id, title
├── style_sample_text (문체 학습 원문)
├── style_analysis_json (분석 결과)
└── automation_level (1~3)

storyteller_characters
├── id, project_id
├── name, brief_description
├── profile_json (AI 생성 전체 프로필)
├── image_asset_id (CMS 캐릭터 이미지 연결)
└── position_x, position_y (관계도 캔버스 위치)

storyteller_relationships
├── id, from_character_id, to_character_id
├── relationship_type
└── memo

storyteller_plot_cards
├── id, project_id
├── stage (발단/전개/위기/절정/결말)
├── title, description
├── character_ids[], status
└── order_index

storyteller_chat_messages
├── id, project_id
├── role (user/assistant)
├── content
├── status (draft/approved)
└── linked_cut_id (CMS 컷 연결)

storyteller_style_versions
├── id, project_id
├── version_number (1, 2, 3 ...)
├── sample_text (이 버전의 학습 원문)
├── analysis_json (어휘 빈도, 문장 길이, 수사법 패턴 등)
├── label (예: "초기 분석", "10화 이후 보정")
├── is_active (현재 사용 중 여부)
└── created_at

storyteller_scenes
├── id, project_id
├── episode_number (화 번호, nullable — 미배정 씬 허용)
├── title (씬 제목 또는 요약)
├── order_index (에피소드 내 순서)
├── status (draft/approved/exported)
├── plot_card_id (연결된 플롯 카드, nullable)
└── message_ids[] (포함된 chat_messages id 목록)

storyteller_plot_branches
├── id, project_id, plot_card_id (기반 플롯 카드)
├── title (분기 제목)
├── description (AI 제안 내용)
├── status (selected / archived / discarded)
│   — selected: 실제 채택된 분기
│   — archived: 선택 안 됐지만 보관 (되살리기 가능)
│   — discarded: 완전 폐기
├── source (ai_suggestion / manual)
└── created_at
```

## 6. 개발 우선순위 (MVP → Full)

### Phase 1: MVP (핵심 가치 검증)
- [ ] 문체 샘플 업로드 + 분석
- [ ] 스토리 채팅 (입력 → AI 확장 → 승인/재생성)
- [ ] 등장인물 카드 생성/편집

### Phase 2: 그래픽 관리
- [ ] 관계도 캔버스 (드래그 연결, 관계 설정)
- [ ] 플롯 보드 (단계별 카드 배치)
- [ ] AI 플롯 제안

### Phase 3: 애셋 통합
- [ ] CMS 캐릭터 이미지 연동
- [ ] 컷 이미지 ↔ 장면 태그
- [ ] 내보내기 (스크립트 포맷, PDF)

### Phase 4: 고도화
- [ ] 자동화 레벨 설정
- [ ] 배경 아카이브
- [ ] 연속 생성 모드

## 7. 차별점

| 기존 AI 글쓰기 도구 | 스토리텔러 |
|---------------------|------------|
| 작가가 AI에게 종속 | 작가가 지휘, AI가 보조 |
| 텍스트만 다룸 | 인물/관계/플롯 시각적 관리 |
| 범용 문체 | 해당 작가의 문체 학습 |
| 결과물이 단편적 | 프로젝트 전체 맥락 유지 |
| 애셋 연동 없음 | CMS 컷/이미지와 완전 통합 |

**핵심 가치**: 작가는 이야기의 방향을 결정하고, AI는 그 방향으로 달린다.

## 8. 프롬프트 엔지니어링 전략

### 8.1 문체 분석 프롬프트 설계

작가 샘플을 받으면 아래 차원을 구조화된 JSON으로 추출한다.

```
[시스템]
당신은 문학 비평가이자 문체 분석 전문가입니다.
주어진 텍스트를 아래 6개 차원으로 분석하고 JSON으로 반환하세요.

[추출 차원]
1. vocabulary_profile
   - avg_sentence_length: 평균 문장 글자 수
   - rhythm_type: "끊어치기" | "호흡 길게" | "혼합"
   - signature_phrases: 자주 쓰는 표현 TOP 10 (빈도 포함)
   - formality_level: 1~5 (1=매우 구어체, 5=문어체)

2. sentence_breathing
   - short_sentence_ratio: 15자 이하 비율
   - paragraph_avg_sentences: 문단당 평균 문장 수
   - uses_incomplete_sentences: 파편 문장 사용 여부 (bool)
   - dash_ellipsis_usage: "—", "…" 빈도 (높음/보통/낮음)

3. rhetorical_patterns
   - metaphor_ratio: 은유 비율 (직유 대비)
   - repetition_usage: 반복 기법 사용 여부 및 패턴
   - contrast_pattern: 대비 구조 자주 쓰는지 여부
   - signature_devices: 특징적 수사법 서술 (자유 텍스트)

4. sensory_description
   - dominant_sense: "시각" | "청각" | "촉각" | "복합"
   - onomatopoeia_density: 의성어/의태어 밀도 (높음/보통/낮음)
   - emotional_interiority_ratio: 내면 묘사 비율 (행동 묘사 대비)

5. dialogue_style
   - speech_ratio: 전체 텍스트 중 대화 비율
   - dialect_or_accent: 사투리/특수 어투 여부
   - tagline_style: 대화 후 서술 방식 ("그가 말했다" 스타일 분류)
   - inner_monologue_style: 내면 독백 처리 방식 (이탤릭/인용/서술 혼합)

6. pacing
   - action_scene_density: 액션 장면 밀도
   - time_skip_pattern: 시간 생략 처리 방식 (장면 전환 문구 패턴)
   - chapter_end_hook: 화 마무리 방식 (여운/클리프행어/전환)
```

### 8.2 스토리 확장 프롬프트 구조

스토리 채팅에서 AI가 한 줄 지시를 완성된 문단으로 확장할 때 사용하는 시스템 프롬프트 조합 구조.

```
[레이어 1 — 문체 프로필]
당신은 {작가명}의 문체로 글을 씁니다.
문체 특징:
- 문장 호흡: {rhythm_type}, 평균 {avg_sentence_length}자

- 수사법: {signature_devices}
- 감각 묘사: {dominant_sense} 중심, 내면 묘사 비율 {emotional_interiority_ratio}
- 대화 처리: {tagline_style}

[레이어 2 — 등장인물 컨텍스트]
현재 등장인물 (이 씬 참여자만 주입):
- {이름}: {성격 요약}, {말투}, {현재 감정 상태}, {이 씬에서의 동기}
- 관계: {씬 참여자 간 관계 한 줄 요약}

[레이어 3 — 현재 플롯 위치]
작품 단계: {발단/전개/위기/절정/결말}
직전 씬 요약: {앞 씬 2~3줄 요약}
현재 씬 목적: {이 씬이 전체 플롯에서 하는 역할}
긴장 수위: {1~10}

[레이어 4 — 작가 지시]
{작가가 입력한 한 줄 지시}
자동화 레벨: {1~3}

→ 위 4개 레이어를 결합해 단일 시스템 프롬프트로 조립 후 호출.
   레이어 2는 씬 참여자 캐릭터만 선별 주입 (전체 캐릭터 목록 ×).
```

### 8.4 토큰 효율 전략 (50화+ 규모)

작품이 장기화될수록 컨텍스트 윈도우가 부족해지는 문제를 3단계 레이어로 관리한다.

```
[요약 레이어 구조]

Level 1 — 프로젝트 요약 (항상 주입, ~500토큰)
  - 작품 전체 시놉시스 1~2단락
  - 핵심 등장인물 3~5명 한 줄 요약
  - 현재 스토리 단계 (절정 진입 중 등)

Level 2 — 에피소드 요약 (최근 5화, ~1000토큰)
  - 각 화 핵심 사건 2~3줄
  - 자동 생성: 화 완료 승인 시 AI가 요약 생성 → storyteller_scenes.summary에 저장

Level 3 — 씬 원문 (현재 작업 씬 전후 3씬, ~2000토큰)
  - 실제 승인된 메시지 원문
  - 직전 씬 끝 문장 ~ 현재 씬 전체

[씬 선별 주입 규칙]
- 현재 씬과 같은 등장인물이 나오는 최근 씬 우선
- 같은 장소/배경 씬 우선
- plot_card 연결이 동일한 씬 우선
- 그 외는 Level 2 요약으로 대체

[캐릭터 컨텍스트 최적화]
- 씬 참여 캐릭터만 프로필 주입 (50화 기준 등장인물 20명 이상 가능)
- 비참여 캐릭터는 이름+한줄요약만 주입 (관계 언급 대비)
```

## 서비스 개요

### 핵심 기능
- **계층적 구조 관리**: 웹툰 → 회차 → 컷 → 파일
- **공정별 뷰**: 공정 중심의 파일 관리
- **파일 관리**: 업로드, 다운로드, 검색, 메타데이터 관리
- **역할 기반 권한**: admin, manager, staff, viewer
- **검색 기능**: 파일 및 콘텐츠 검색
- **협업 도구**: 다중 사용자 작업 관리

### 사용자 시나리오
1. **작가/매니저**: 웹툰/회차/컷 생성 및 관리
2. **스태프**: 파일 업로드 및 작업 진행
3. **관리자**: 전체 시스템 관리 및 사용자 관리
4. **조회자**: 콘텐츠 조회 및 파일 다운로드

## 벤치마킹 서비스 분석

### 1. Notion (노션)

**선정 이유**
- 계층적 구조 관리 (페이지 → 블록 → 콘텐츠)
- 사이드바 기반 네비게이션
- 깔끔하고 미니멀한 UI
- 강력한 검색 기능

**주요 디자인 특징**
- **색상**: 흰색 배경, 회색 계열의 부드러운 색상 팔레트
- **타이포그래피**: San Francisco (SF Pro) 계열, 명확한 계층 구조
- **레이아웃**: 왼쪽 사이드바 + 메인 콘텐츠 영역
- **인터랙션**: 호버 시 미묘한 배경색 변화, 부드러운 트랜지션
- **컴포넌트**: 카드 기반 레이아웃, 아이콘 + 텍스트 조합

**적용 포인트**
- 사이드바 네비게이션 구조
- 카드 기반 목록 표시
- 부드러운 호버 효과
- 명확한 계층 구조 시각화

### 2. Linear (리니어)

**선정 이유**
- 프로젝트 관리 도구
- 탭 기반 네비게이션
- 미니멀하고 빠른 UI
- 키보드 단축키 중심

**주요 디자인 특징**
- **색상**: 다크 모드 중심, 높은 대비
- **타이포그래피**: Inter 폰트, 작은 크기로도 가독성 우수
- **레이아웃**: 상단 탭 + 메인 영역, 사이드바 최소화
- **인터랙션**: 즉각적인 피드백, 애니메이션 최소화
- **컴포넌트**: 배지, 태그, 상태 표시기

**적용 포인트**
- 빠른 인터랙션 피드백
- 상태 표시 (배지, 태그)
- 키보드 네비게이션
- 미니멀한 디자인

### 3. Figma (피그마)

**선정 이유**
- 파일 관리 및 그리드 레이아웃
- 협업 기능
- 이미지 미리보기 중심
- 직관적인 파일 탐색

**주요 디자인 특징**
- **색상**: 밝은 배경, 강조색 사용
- **타이포그래피**: Inter, 명확한 라벨링
- **레이아웃**: 그리드 기반 파일 표시
- **인터랙션**: 드래그 앤 드롭, 컨텍스트 메뉴
- **컴포넌트**: 썸네일 그리드, 파일 카드

**적용 포인트**
- 파일 그리드 레이아웃
- 이미지 썸네일 표시
- 드래그 앤 드롭 업로드
- 파일 카드 디자인

## 디자인 원칙

### 1. 명확성 (Clarity)
- 모든 정보와 기능이 명확하게 표시되어야 함
- 불필요한 요소 제거, 핵심 기능 강조
- 일관된 아이콘과 라벨 사용

### 2. 효율성 (Efficiency)
- 자주 사용하는 기능에 빠르게 접근 가능
- 키보드 단축키 지원
- 최소한의 클릭으로 작업 완료

### 3. 일관성 (Consistency)
- 동일한 패턴의 반복 사용
- 일관된 색상, 타이포그래피, 간격
- 예측 가능한 인터랙션

### 4. 접근성 (Accessibility)
- WCAG 2.1 AA 기준 준수
- 키보드 네비게이션 지원
- 색상 대비 비율 준수
- 스크린 리더 지원

### 5. 확장성 (Scalability)
- 다양한 화면 크기 지원
- 많은 데이터 처리 가능
- 새로운 기능 추가 용이

## 색상 시스템

### 기본 색상 팔레트

#### Primary (주요 색상)
```css
--primary: oklch(0.205 0 0);           /* #333333 - 거의 검은색 */
--primary-foreground: oklch(0.985 0 0); /* #FBFBFB - 거의 흰색 */
```
- **용도**: 주요 버튼, 링크, 강조 요소
- **사용 예**: "생성", "저장", "확인" 버튼

#### Secondary (보조 색상)
```css
--secondary: oklch(0.97 0 0);          /* #F7F7F7 - 매우 밝은 회색 */
--secondary-foreground: oklch(0.205 0 0); /* #333333 */
```
- **용도**: 보조 버튼, 배경, 구분선
- **사용 예**: "취소", "뒤로" 버튼

#### Accent (강조 색상)
```css
--accent: oklch(0.97 0 0);             /* #F7F7F7 */
--accent-foreground: oklch(0.205 0 0);  /* #333333 */
```
- **용도**: 호버 상태, 선택된 항목
- **사용 예**: 목록 항목 호버, 탭 선택

#### Muted (약한 색상)
```css
--muted: oklch(0.97 0 0);              /* #F7F7F7 */
--muted-foreground: oklch(0.556 0 0);  /* #8E8E8E */
```
- **용도**: 부가 정보, 플레이스홀더
- **사용 예**: 설명 텍스트, 비활성 상태

#### Destructive (경고/삭제 색상)
```css
--destructive: oklch(0.577 0.245 27.325); /* #DC2626 - 빨간색 */
```
- **용도**: 삭제, 경고, 오류 메시지
- **사용 예**: "삭제" 버튼, 오류 알림

#### Border (경계선)
```css
--border: oklch(0.922 0 0);            /* #EBEBEB - 밝은 회색 */
```
- **용도**: 카드, 입력 필드, 구분선
- **사용 예**: 카드 테두리, 입력 필드 테두리

### 다크 모드 색상

다크 모드에서는 색상의 명도와 채도를 조정하여 가독성을 유지합니다.

```css
.dark {
  --background: oklch(0.145 0 0);      /* #252525 - 어두운 배경 */
  --foreground: oklch(0.985 0 0);     /* #FBFBFB - 밝은 텍스트 */
  --card: oklch(0.205 0 0);           /* #333333 - 카드 배경 */
  --border: oklch(1 0 0 / 10%);        /* 반투명 흰색 */
}
```

### 색상 사용 가이드라인

#### Notion 스타일: 부드러운 색상 팔레트
1. **회색 계열의 미묘한 색상 변화**: Muted 색상을 활용한 부드러운 배경
2. **호버 시 미묘한 배경색 변화**: Accent 색상으로 부드럽게 전환 (transition-colors duration-150)
3. **선택된 항목**: Primary 색상 배경 또는 ring으로 명확히 표시
4. **테두리**: 얇고 부드러운 Border 색상 사용

#### Linear 스타일: 높은 대비
1. **다크 모드에서의 색상 대비**: 텍스트와 배경 간 최소 4.5:1 대비 유지
2. **명확한 상태 표시**: 배지와 태그에 색상으로 구분
3. **Primary는 신중하게 사용**: 주요 액션에만 사용
4. **일관된 색상 사용**: 동일한 의미는 동일한 색상 사용

## 타이포그래피

### 폰트 패밀리

```css
--font-sans: var(--font-geist-sans);   /* Geist Sans */
--font-mono: var(--font-geist-mono);   /* Geist Mono */
```

- **Sans**: 본문, UI 요소
- **Mono**: 코드, 파일명, 기술적 정보

### 타이포그래피 스케일

#### 제목 (Headings)
```css
h1: text-2xl / font-bold      /* 24px / 700 */
h2: text-xl / font-semibold   /* 20px / 600 */
h3: text-lg / font-semibold   /* 18px / 600 */
h4: text-base / font-medium   /* 16px / 500 */
```

#### 본문 (Body)
```css
body: text-base / font-normal     /* 16px / 400 */
small: text-sm / font-normal      /* 14px / 400 */
caption: text-xs / font-normal    /* 12px / 400 */
```

#### UI 요소
```css
button: text-sm / font-medium     /* 14px / 500 */
label: text-sm / font-medium      /* 14px / 500 */
input: text-sm / font-normal      /* 14px / 400 */
```

### 타이포그래피 사용 가이드라인

1. **계층 구조 명확히**: 제목 > 본문 > 캡션
2. **가독성 우선**: 최소 14px 이상 사용
3. **일관된 간격**: line-height 1.5~1.6 권장
4. **강조는 신중하게**: Bold는 제목과 중요 정보에만

## 레이아웃 및 그리드

### 기본 레이아웃 구조

```
┌─────────────────────────────────────────┐
│           Navigation Bar            │
├─────────────────────────────────────────┤
│                                         │
│  ┌──────────┐  ┌──────────┐  ┌────────┐│
│  │          │  │          │  │        ││
│  │ Sidebar  │  │  Main    │  │ Detail ││
│  │  (240px) │  │ Content  │  │  Panel ││
│  │          │  │          │  │        ││
│  └──────────┘  └──────────┘  └────────┘│
│                                         │
└─────────────────────────────────────────┘
```

### 그리드 시스템

- **컨테이너 최대 너비**: 1280px (max-w-7xl)
- **패딩**: 좌우 16px (모바일), 24px (데스크톱)
- **간격**: 8px 단위 (8, 16, 24, 32, 40, 48px)
- **공백 활용**: 충분한 여백으로 가독성 향상, 카드 간 간격 16px (gap-4), 섹션 간 간격 24px (gap-6)

### 사이드바 레이아웃 (Notion 스타일)

모든 사이드바는 **240px 고정 너비**를 사용하여 일관성을 유지합니다.

#### 사이드바 공통 스타일
- **너비**: 240px 고정 (w-60)
- **테두리**: 오른쪽 얇은 테두리 (border-r, border 색상)
- **배경**: 부드러운 배경색 (bg-sidebar 또는 bg-muted)
- **스크롤**: 각 패널 독립 스크롤 (ScrollArea 사용)
- **호버 효과**: 목록 항목 호버 시 accent 배경색으로 부드럽게 전환 (150ms ease-in-out)

#### 웹툰별 뷰 (4단 패널 - Notion 스타일)

데스크톱에서 4개 패널을 나란히 배치하여 계층 구조를 한눈에 볼 수 있도록 합니다.

```
┌─────────┬─────────┬─────────┬──────────────┐
│ 웹툰    │ 회차    │ 컷      │ 파일 그리드  │
│ (240px) │ (240px) │ (240px) │ (flex)       │
│         │         │         │              │
│ [목록]  │ [목록]  │ [목록]  │ [그리드]     │
│         │         │         │              │
└─────────┴─────────┴─────────┴──────────────┘
```

**구현 세부사항**
- **각 패널 너비**: 240px 고정
- **패널 간 구분**: 얇은 수직선 (border-r, border 색상)
- **스크롤**: 각 패널 독립 스크롤 (ScrollArea)
- **선택 표시**: 선택된 항목은 Primary 색상 배경 또는 ring-2 ring-primary
- **호버 효과**: 목록 항목 호버 시 accent 배경색 (transition-colors duration-150)
- **패딩**: 패널 내부 좌우 16px (px-4)

**반응형 동작**
- **모바일 (< 1024px)**: 세로 스택 레이아웃, 한 번에 하나의 패널만 표시
- **데스크톱 (≥ 1024px)**: 4개 패널 나란히 배치

#### 공정별 뷰 (2단 패널)

```
┌─────────┬─────────────────────────────┐
│ 공정    │ 파일 그리드                 │
│ (240px) │ (flex)                      │
│         │                             │
│ [목록]  │ [그리드]                    │
│         │                             │
└─────────┴─────────────────────────────┘
```

**구현 세부사항**
- **공정 패널 너비**: 240px 고정
- **파일 그리드**: 나머지 공간 사용 (flex-1)
- **구분선**: 얇은 수직선 (border-r)
- **스크롤**: 각 영역 독립 스크롤

### 반응형 브레이크포인트

```css
sm: 640px   /* 태블릿 세로 */
md: 768px   /* 태블릿 가로 */
lg: 1024px  /* 데스크톱 */
xl: 1280px  /* 큰 데스크톱 */
```

## 컴포넌트 디자인

### 버튼 (Button)

#### Variants
- **default**: Primary 색상, 주요 액션
- **outline**: 테두리만, 보조 액션
- **ghost**: 배경 없음, 텍스트만
- **destructive**: 빨간색, 삭제/위험 액션

#### Sizes
- **sm**: 높이 32px, 패딩 8px 16px
- **default**: 높이 40px, 패딩 12px 24px
- **lg**: 높이 48px, 패딩 16px 32px

#### 예시
```tsx
<Button variant="default" size="default">
  웹툰 생성
</Button>
<Button variant="outline" size="sm">
  취소
</Button>
<Button variant="destructive" size="sm">
  삭제
</Button>
```

### 카드 (Card)

#### 구조
- **배경**: 흰색 (다크 모드: 어두운 회색)
- **테두리**: 1px, Border 색상
- **패딩**: 16px
- **둥근 모서리**: 8px (radius-md)
- **그림자**: 호버 시 미묘한 그림자

#### 사용 예시
```tsx
<Card className="hover:shadow-md transition-shadow cursor-pointer">
  <CardHeader>
    <CardTitle>웹툰 제목</CardTitle>
  </CardHeader>
  <CardContent>
    <p>설명 텍스트</p>
  </CardContent>
</Card>
```

### 입력 필드 (Input)

#### 스타일
- **높이**: 40px
- **패딩**: 좌우 12px
- **테두리**: 1px, Border 색상
- **둥근 모서리**: 6px
- **포커스**: Primary 색상 링

#### 상태
- **기본**: Border 색상
- **포커스**: Primary 색상 링
- **에러**: Destructive 색상
- **비활성**: Muted 배경

### 다이얼로그 (Dialog)

#### 구조
- **오버레이**: 반투명 검은색 (rgba(0,0,0,0.5))
- **패널**: 중앙 정렬, 최대 너비 500px
- **패딩**: 24px
- **둥근 모서리**: 12px

#### 사용 예시
```tsx
<Dialog>

  <DialogContent>
    <DialogHeader>

    </DialogBody>
```

### 배지 (Badge)

#### Variants
- **default**: Primary 배경
- **secondary**: Secondary 배경
- **outline**: 테두리만
- **destructive**: Destructive 배경

#### 사용 예시
```tsx
<Badge variant="default">관리자</Badge>
```

### 파일 그리드 (File Grid)

#### 그리드 레이아웃
- **열 수**: 반응형 (모바일 2열, 태블릿 3열, 데스크톱 4열)
- **간격**: 16px
- **카드 크기**: 정사각형 비율 유지

#### 파일 카드 (Figma 스타일)
- **썸네일**: 상단, 16:9 비율 유지
- **정보**: 하단에 간결한 정보 (파일명, 공정, 날짜)
- **액션**: 호버 시 표시 (다운로드, 삭제)
- **레이아웃**: 균등한 간격의 그리드 (gap-4)
- **스타일**: 미니멀한 테두리, 충분한 여백
- **호버 효과**: 그림자 증가 (shadow-sm → shadow-md)

## 네비게이션 패턴

### 상단 네비게이션 바

**구조**
```
[로고/제목] [뷰 전환] [검색] [사용자 메뉴]
```

**요소**
- **로고/제목**: 왼쪽, 클릭 시 홈으로 이동
- **뷰 전환**: 중앙 왼쪽, 웹툰별/공정별 전환
- **검색**: 중앙, 전체 너비의 40%
- **사용자 메뉴**: 오른쪽, 드롭다운

### 사이드바 네비게이션 (Notion 스타일)

#### 웹툰별 뷰 (4단 패널)
- **레이아웃**: 데스크톱에서 4개 패널 나란히 배치
- **각 패널 너비**: 240px 고정
- **스크롤**: 각 패널 독립 스크롤 (ScrollArea)
- **선택 표시**: Primary 색상 배경 또는 ring-2 ring-primary
- **호버 효과**: 목록 항목 호버 시 accent 배경색으로 부드럽게 전환
- **구분선**: 패널 간 얇은 수직선 (border-r)

#### 공정별 뷰 (2단 패널)
- **레이아웃**: 공정 목록(240px) + 파일 그리드(flex)
- **공정 패널 너비**: 240px 고정
- **스크롤**: 각 영역 독립 스크롤
- **순서 변경**: 화살표 버튼으로 공정 순서 조정

### 브레드크럼 네비게이션

**구조**
```
홈 > 웹툰명 > 회차명 > 컷명
```

**사용 예시**
- 현재 위치 표시
- 상위 레벨로 빠른 이동
- 검색 결과에서 컨텍스트 제공

- **선택된 항목**: Primary 색상 배경 또는 ring-2 ring-primary

- **Active 상태**: 약간 눌린 효과 (scale: 0.98)
- **트랜지션**: 100ms

#### 카드 클릭
- **리플 효과**: 클릭 지점에서 확산
- **트랜지션**: 300ms

### 로딩 상태

#### 스켈레톤 UI
- **애니메이션**: 펄스 효과
- **색상**: Muted 배경
- **사용 위치**: 목록, 카드, 그리드

#### 스피너
- **크기**: 24px
- **색상**: Primary
- **위치**: 중앙 정렬

### 에러 상태

#### 에러 메시지
- **색상**: Destructive
- **아이콘**: AlertCircle
- **위치**: 입력 필드 아래 또는 상단 배너

- **아이콘**: 큰 크기, Muted 색상
- **메시지**: 명확한 안내

## 시각적 디자인 원칙

### 공백(Whitespace) 활용

충분한 여백을 활용하여 가독성과 시각적 계층을 명확히 합니다.

#### 간격 규칙
- **카드 간 간격**: 16px (gap-4)
- **섹션 간 간격**: 24px (gap-6)
- **패딩**: 카드 내부 16px (p-4), 섹션 내부 24px (p-6)
- **여백**: 불필요한 요소 제거, 핵심 콘텐츠에 집중

#### Notion 스타일 공백
- **목록 항목**: 상하 패딩 8px (py-2)
- **카드**: 내부 패딩 16px (p-4)
- **섹션**: 상하 여백 24px (my-6)

### 계층 구조 시각화

명확한 시각적 계층을 통해 정보의 중요도를 표현합니다.

#### 타이포그래피 계층
- **제목**: 큰 크기, 굵은 글꼴 (text-2xl, font-bold)
- **부제목**: 중간 크기, 세미볼드 (text-xl, font-semibold)
- **본문**: 기본 크기, 일반 글꼴 (text-base, font-normal)
- **캡션**: 작은 크기, Muted 색상 (text-sm, text-muted-foreground)

#### 색상 계층
- **Primary**: 가장 중요한 액션과 선택된 항목
- **Foreground**: 주요 텍스트
- **Muted Foreground**: 부가 정보, 설명 텍스트
- **Border**: 구분선, 테두리

#### 아이콘과 텍스트 균형
- **아이콘 크기**: 텍스트와 조화롭게 (h-4 w-4 ~ h-6 w-6)
- **아이콘 + 텍스트**: gap-2로 적절한 간격 유지
- **아이콘만**: 명확한 aria-label 제공

- **성공**: 미묘한 성공 표시 (배지 또는 토스트)

- **배지**: 작고 명확한 배지 (text-xs, 적절한 variant)
- **태그**: 색상으로 구분 (공정별 색상, 상태별 색상)

#### 높은 대비
- **다크 모드**: 텍스트와 배경 간 최소 4.5:1 대비 유지
- **명확한 상태**: 색상으로 상태를 즉시 인식 가능

- **트랜지션**: 부드러운 애니메이션 (200ms ease-in-out)

- **구조**: 썸네일(상단) + 정보(하단)

- **액션 버튼**: 호버 시 표시 (다운로드, 삭제, 수정)
- **패딩**: 내부 패딩 16px (p-4)

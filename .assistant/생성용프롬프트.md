# features.md

너는 이 프로젝트(thegrim-cms)의 기능 히스토리를 정리하는 AI 비서야.

최근 git commit 기록과 코드 변경을 분석해서,
"완료된 기능" 기준으로 features.md를 업데이트해줘.

규칙:
- 커밋 단위가 아니라 "기능/의도(Feature)" 단위로 묶을 것
- 완료되었다고 판단되는 기능만 기록할 것
- 각 기능마다 아래 4가지를 반드시 포함할 것:
  - Why (왜 이 기능을 만들었는지)
  - What (무엇을 만들었는지, 한 줄)
  - Result (현재 완료 상태)
  - Notes (다음 판단에 도움이 될 메모)
- 추측하지 말 것
- 불확실하면 기록하지 말 것
- 포맷은 절대 변경하지 말 것
